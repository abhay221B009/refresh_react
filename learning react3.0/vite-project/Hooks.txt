1.Side effects:
operations that reach outside the functional scope of a react component .these operations can affect the other component, interact with the browser, or services or perform asynchronous data fetching.
anythings that are related to rendering 
e.g: setTimeout, fetch, setInterval, document .getElementbyId

2.Hooks:
Hooks are special functions in React that let you use state and lifecycle features(without using or writing class) in functional components (without needing class components).
useState
 useEffect
 useCallback
 useMemo
 useRef
 useContext


 useState is a React Hook that lets you add and manage state in functional components.


 useEffect is used to handle side effects in functional components.
👉 Side effects = API calls, timers, DOM updates, subscriptions, etc.


useMemo is a React Hook used to memoize (cache) the result of a calculation so it doesn’t get recomputed on every render unless its dependencies change.


useCallback is a hook in react , a popular javascript library for building user interfaces. it is used to memoize functions , which can help in optimizing the performance of your application, especially in cases involving child components that rely on reference equality to prevent unncescessary renders


A custom hook is just a function that uses React hooks (useState, useEffect, etc.) and lets you reuse logic.

👉 Instead of repeating logic in multiple components, you extract it.c
